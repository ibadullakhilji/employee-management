3. DTO Layer (Data Transfer Object)
Right now your API directly exposes your Employee entity to the outside world. This is bad in real applications. Let me explain why:

The Problem:
Imagine your Employee entity has sensitive fields like:
password;
private String ssn;          // Social Security Number
private String bankAccount;

    **The Solution — DTO:**
    - **Request DTO** → Controls what data the **client can send IN**
    - **Response DTO** → Controls what data the **client gets OUT**
    - Your entity stays **safe and hidden** inside your backend

    Architecture
    Client → RequestDTO → Service → Entity → Database
    Database → Entity → Service → ResponseDTO → Client


#4 — Global Exception Handler?
Right now if something goes wrong, Spring returns ugly error responses,
        After implementing Global Exception Handler, you'll get clean, meaningful responses
        How it works:
        Request comes in → Something goes wrong → Exception is thrown →
        Global Handler catches it → Returns clean error response

     Instead of handling errors in every single method, we create ONE central place that handles all errors for the entire application!

7. JWT Security 🔐
Right now anyone can access your API — no login, no authentication. JWT Security adds:

        User Registration & Login
        Password encryption
        JWT Token generation
        Protected endpoints — only logged in users can access

How JWT works:
        1. User registers → password gets encrypted → saved to DB
        2. User logs in → server verifies password → generates JWT token
        3. User sends token with every request → server validates token → allows access
